# Multi-language Support (i18n) Design

## Goal

Add Chinese (zh-CN) + English language support to the Image Eval Pipeline UI and pipeline stage messages, with a sidebar language toggle.

## Scope

- All user-facing text in `app.py` (headers, buttons, labels, status messages, error text, dimension labels, critique transcript labels, expander titles, empty state)
- Pipeline stage messages in `pipeline.py` (the `on_stage()` callback strings)
- Language switcher in the Streamlit sidebar

## Approach: Dict-based i18n

A single `i18n.py` module with a flat translation dict per language and a `t(key)` lookup function. No external dependencies.

### New file: `i18n.py`

```python
import streamlit as st

TRANSLATIONS = {
    "en": { ... },
    "zh": { ... },
}

def get_lang() -> str:
    return st.session_state.get("lang", "zh")

def t(key: str, **kwargs) -> str:
    lang = get_lang()
    text = TRANSLATIONS.get(lang, TRANSLATIONS["en"]).get(key, key)
    if kwargs:
        text = text.format(**kwargs)
    return text
```

### Language switcher

In the Streamlit sidebar:
```python
lang_options = {"English": "en", "中文": "zh"}
selected = st.sidebar.selectbox(t("lang_label"), list(lang_options.keys()), ...)
st.session_state.lang = lang_options[selected]
```

Default language: `"zh"`.

### Changes by file

| File | Change |
|------|--------|
| `i18n.py` | New file — translations dict + `t()` function |
| `app.py` | Import `t`, add sidebar switcher, replace all hardcoded strings with `t()` calls |
| `pipeline.py` | Import `t`, replace `notify()` string literals with `t()` calls |

### What stays unchanged

- CSS class names and styling
- Schema/model code (`schemas.py`, `compare.py`, `evaluate.py`, `critique.py`, `revise.py`, `generate.py`)
- Plotly chart configuration (axis ticks use dimension labels which will be translated via `t()`)
- Test files

### Default language

`"zh"` — the primary user operates in Chinese. English is the secondary language.
